# i18n — Babel/gettext workflow

Бот поддерживает два языка: `ru` (по умолчанию) и `en`. Перевод выполняется
через стандартный Python `gettext` (runtime) + Babel (build-time CLI для
извлечения/обновления/компиляции переводов).

Покрытие переводом: пользовательские handler'ы (`app/handlers/*.py`),
клавиатуры (`app/keyboards/*.py`), публичные шаблоны (`app/templates/setup_landing.html`,
`app/templates/public_*.html`). **Admin-панель (`app/templates/admin_*.html`)
остаётся только на ru** — внутренний UI для команды поддержки.

## Структура

```
babel.cfg                         # маппинг extract'а: python + jinja2
locale/
  messages.pot                    # шаблон (генерируется, не коммитится)
  en/LC_MESSAGES/
    messages.po                   # english переводы (коммитится)
    messages.mo                   # скомпилированные (коммитятся для prod)
  ru/LC_MESSAGES/
    messages.po
    messages.mo
```

## Команды

Установить Babel: уже в `requirements.txt` (build/dev зависимость).

### Извлечь строки из кода в шаблон

```bash
pybabel extract -F babel.cfg -o locale/messages.pot app/
```

### Создать .po для нового языка (первый раз)

```bash
pybabel init -i locale/messages.pot -d locale -l <lang>
```

### Обновить существующие .po (после нового extract)

```bash
pybabel update -i locale/messages.pot -d locale
```

Это сохраняет существующие переводы и помечает изменённые строки как
`fuzzy` — их нужно перепроверить и убрать пометку.

### Скомпилировать .po → .mo

```bash
pybabel compile -d locale
```

Runtime читает только `.mo` — `.po` без компиляции не работают.

## Полный цикл при добавлении новой строки

1. Обернуть строку в коде: `_('Текст')` (callable приходит через middleware,
   см. [`app/middlewares/i18n.py`](../app/middlewares/i18n.py)).
2. `pybabel extract -F babel.cfg -o locale/messages.pot app/`
3. `pybabel update -i locale/messages.pot -d locale`
4. Перевести в `locale/en/LC_MESSAGES/messages.po` и `locale/ru/LC_MESSAGES/messages.po`.
5. `pybabel compile -d locale`
6. Закоммитить изменения `.po` и `.mo` (см. ниже).

## Что коммитим

- `babel.cfg` — конфиг extract'а.
- `locale/<lang>/LC_MESSAGES/messages.po` — текстовые переводы (human-readable).
- `locale/<lang>/LC_MESSAGES/messages.mo` — скомпилированный binary, читается
  в runtime; коммитим, чтобы Docker-образ работал out-of-the-box без
  pybabel в build-цепочке.

`.pot` НЕ коммитим — это промежуточный артефакт, регенерируется через extract.
